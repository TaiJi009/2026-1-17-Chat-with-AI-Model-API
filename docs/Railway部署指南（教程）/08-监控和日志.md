# 监控和日志

Railway 提供了丰富的监控和日志功能，帮助您了解应用的运行状态。

## 查看实时日志

### 方法 1: 使用 Railway 控制台

1. 在服务页面点击 **"Deployments"** 标签
2. 找到最新的部署，点击 **"View Logs"**
3. 可以查看实时应用日志

### 方法 2: 使用 Railway CLI

```bash
# 查看实时日志
railway logs -f

# 查看特定服务的日志
railway logs --service backend
```

## 日志类型

Railway 提供以下类型的日志：

- **Build Logs**: 构建过程的日志
- **Deploy Logs**: 部署过程的日志
- **Runtime Logs**: 应用运行时的日志

## 查看指标

Railway 提供基本的监控指标：

- **CPU 使用率**: 服务的 CPU 使用情况
- **内存使用率**: 服务的内存使用情况
- **网络流量**: 服务的网络流量统计
- **请求数**: 服务的请求数量

在服务页面的 **Metrics** 标签可以查看。

## 日志管理

### 日志保留

- Railway 会保留最近一段时间的日志
- 对于长期日志存储，建议使用外部日志服务

### 外部日志服务

如果需要长期存储日志，可以集成以下服务：

- **Logtail**: 实时日志收集和分析
- **Datadog**: 应用监控和日志管理
- **Sentry**: 错误追踪和日志管理

## 监控最佳实践

1. **定期查看日志**：
   - 检查错误和警告信息
   - 监控应用性能

2. **设置告警**：
   - 使用外部监控服务设置告警
   - 监控关键指标

3. **日志分析**：
   - 使用日志分析工具
   - 识别性能瓶颈

## 常见日志查看场景

### 查看构建日志

如果部署失败，查看构建日志：

1. 在 **Deployments** 页面
2. 找到失败的部署
3. 点击 **"View Logs"**
4. 查看 **Build Logs** 标签

### 查看运行时日志

查看应用运行时的日志：

1. 在服务页面
2. 点击 **"Logs"** 标签
3. 查看实时日志

### 搜索日志

在日志页面可以使用搜索功能：

- 搜索特定关键词
- 过滤日志级别
- 按时间范围筛选
