# 提示词工程 (Prompt Engineering)

本项目用于管理和维护 AI Agent 大模型相关的提示词工程文档和实践。

## 📋 目录结构

```
prompt-engineering/
├── README.md                    # 本文件，项目介绍
├── Prompt-Writing.md            # 提示词工程结构化文本
├── best-practices.md            # 提示词工程最佳实践
└── templates/                   # 提示词模板库
    └── README.md                # 模板使用说明
```

## 🎯 用途

- 📝 **整理和优化系统提示词**：记录和改进项目中使用的高质量系统提示词
- 🔄 **记录有效的提示词模板**：保存可复用的提示词模板和模式
- 📚 **积累提示词工程最佳实践**：总结经验和教训，提升提示词质量
- 🤖 **为 AI Agent 提供标准化参考**：建立统一的提示词规范和标准

## 🔗 相关项目文件

本项目的提示词相关功能位于：

- `src/components/PromptPanel.tsx` - 提示词配置面板组件
- `src/utils/promptTemplate.ts` - 提示词模板处理工具
- `src/contexts/AppContext.tsx` - 提示词配置状态管理

### 支持的模板变量

- `{{user_input}}` - 用户输入的消息
- `{{date}}` - 当前日期
- `{{time}}` - 当前时间

## 📖 使用指南

### 如何添加新的提示词

1. 在 `Prompt-Writing.md` 中记录提示词的结构化描述
2. 在 `templates/` 文件夹中保存可复用的模板
3. 在 `best-practices.md` 中记录相关的经验和最佳实践

### 提示词测试流程

1. 在项目中通过 `PromptPanel` 组件测试新提示词
2. 记录测试结果和效果评估
3. 将有效的提示词整理到文档和模板库中
