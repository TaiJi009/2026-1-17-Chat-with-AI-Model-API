# 提示词工程最佳实践

本文档记录提示词工程的经验总结和最佳实践。

## 📝 基本准则

### 1. 清晰明确
- ✅ 使用具体、明确的指令
- ❌ 避免模糊和歧义的表达

### 2. 结构化组织
- ✅ 使用列表、分段、标题等方式组织内容
- ✅ 将复杂任务分解为多个步骤

### 3. 上下文提供
- ✅ 提供足够的背景信息和上下文
- ✅ 明确说明期望的输出格式

### 4. 角色定义
- ✅ 明确定义 AI 的角色和职责
- ✅ 说明 AI 应该具备的能力和特点

## 🎯 系统提示词设计

### 角色设定
```
你是[角色描述]，具备[能力1]、[能力2]等特点。
```

### 行为规范
```
你的回答应该：
- 点1
- 点2
- 点3
```

### 输出格式
```
请使用以下格式回答：
1. 标题
2. 详细内容
3. 总结
```

## 📋 模板变量使用

### 常用变量
- `{{user_input}}` - 用户输入内容
- `{{date}}` - 当前日期
- `{{time}}` - 当前时间

### 使用建议
- 变量命名要语义清晰
- 在模板中明确说明变量的用途
- 提供变量的示例值

## ⚠️ 常见问题

### 问题1：提示词过长导致效果下降
**解决方案**：
- 精简冗余内容
- 使用结构化格式提高可读性
- 将复杂指令分解为多个部分

### 问题2：模型理解偏差
**解决方案**：
- 使用具体示例说明期望行为
- 提供正反例对比
- 明确边界条件和约束

### 问题3：输出格式不一致
**解决方案**：
- 明确指定输出格式（JSON、Markdown等）
- 提供格式示例
- 使用结构化指令

## 🔍 测试和优化

### 测试流程
1. **单次测试**：验证基本功能
2. **批量测试**：检查不同场景下的表现
3. **边界测试**：测试极端情况和异常输入
4. **用户反馈**：收集实际使用中的反馈

### 优化策略
- A/B 测试不同版本的提示词
- 记录和分析失败案例
- 持续迭代和改进
